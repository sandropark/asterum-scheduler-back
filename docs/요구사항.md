# 아스테룸 통합 스케줄러 (Asterum Integrated Scheduler)

1. 개요 (Scenario)
   아티스트의 성공적인 활동을 위해서는 아티스트의 스케줄뿐만 아니라 기술팀, 촬영팀, 디자인팀 등 수많은 유관 부서가 하나의 타임라인 위에서 유기적으로 움직여야 합니다.

단순히 개인의 일정을 기록하는 것을 넘어, 아티스트와 제작진 모두가 실시간으로 소통하고 변화하는 촬영 변수에 대응할 수 있는 시스템이 필요합니다. 아스테룸의 모든 구성원이 완벽한 합을 맞출 수 있도록 견고한 스케줄러를 설계하고 구현해 주세요.

2. 구현 범위 (User Stories)
   본 과제는 **풀스택 구현을 원칙**으로 하며, 프론트엔드(UI)와 백엔드(API & DB)가 유기적으로 동작해야 합니다.

🔹 MVP 1: 일회성 촬영 일정
- 사용자로서, 나는 **월간 달력에서 특정 날짜를 선택**해 촬영이나 연습 **일정을 등록**하고 싶어요.
- 사용자로서, **일정 등록 시 제목, 시간과 더불어 해당 일정에 참여하는 사람(멤버 또는 스태프)을 리스트에서 선택하여 등록**하고 싶어요. (참여자 데이터는 CRUD를 직접 구현하거나, 초기 시드 데이터를 활용해 주세요.)
- 사용자로서, 등록된 일정을 클릭하여 **상세 내용**을 확인하고 **수정**하거나 **삭제**할 수 있어야 해요.

🔹 MVP 2: **반복**되는 촬영 일정
- 사용자로서, '매주 월요일 정기 기술 회의'나 '매일 오전 멤버 트레이닝'처럼 반복되는 일정을 등록하고 싶어요.
  - 반복 옵션: 매일, 매주, 매월
  - 종료 조건: 특정 날짜까지 반복 / 특정 횟수 반복 / 무기한 반복
  - 사용자로서, 반복 일정 중 특정 날짜의 일정만 수정하거나 삭제할 수 있어야 합니다.
  - 사용자로서, 반복 일정의 제목이나 시간을 변경할 때 "이 일정만 / 이후 모든 일정 / 전체 일정" 중 범위를 선택하여 일괄 변경하고 싶어요.

🚀 확장 기능 (선택 사항)
- 유형 전환: 일회성 미팅이 정기 점검으로 확정될 경우, 기존 단일 일정을 반복 일정의 첫 번째 시작점으로 즉시 전환하고 싶어요.
- 팀 단위 지정: 개별 사람을 추가하는 대신 '영상팀', '안무팀' 등 특정 팀을 통째로 참여자로 지정하고 싶어요. (팀-멤버 관계 모델링 포함)
- 리소스 중복 방지: 일정 등록 시 촬영 장소(예: 스튜디오 A)를 선택할 수 있으며, 동일한 시간에 **중복 예약이 발생하지 않도록(Race Condition 고려) 방어** 로직이 필요합니다.
- 영속성 유지: 모든 데이터는 실제 데이터베이스와 연동되어 서버나 브라우저를 재시작해도 안전하게 보존되어야 합니다.
